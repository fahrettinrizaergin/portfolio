### 1. Tables (Relations) ğŸ“Š
Tables are the building blocks of an RDBMS. Let's look at a simple example:

```sql
CREATE TABLE products (
    id INT PRIMARY KEY,
    name VARCHAR(100),
    price DECIMAL(10,2),
    stock_quantity INT
);
```

### 2. Relationships ğŸ”—
RDBMS allows us to create connections between tables:
- One-to-One (1:1)
- One-to-Many (1:N)
- Many-to-Many (M:N)

Example of a relationship:
```sql
CREATE TABLE orders (
    id INT PRIMARY KEY,
    customer_id INT,
    order_date DATE,
    total_amount DECIMAL(10,2),
    FOREIGN KEY (customer_id) REFERENCES customers(id)
);
```

### 2. SQL Support ğŸ’»
RDBMS uses SQL (Structured Query Language) for:
```sql
-- Creating data
INSERT INTO products (name, price, stock_quantity)
VALUES ('Smartphone', 599.99, 100);

-- Reading data
SELECT name, price
FROM products
WHERE stock_quantity > 0;

-- Updating data
UPDATE products
SET price = 549.99
WHERE name = 'Smartphone';

-- Deleting data
DELETE FROM products
WHERE stock_quantity = 0;
```

## Database Normalization ğŸ“

Normalization helps organize data efficiently:

1. **First Normal Form (1NF)**
   - No repeating groups
   - Each cell contains a single value

2. **Second Normal Form (2NF)**
   - Meets 1NF requirements
   - No partial dependencies

3. **Third Normal Form (3NF)**
   - Meets 2NF requirements
   - No transitive dependencies
