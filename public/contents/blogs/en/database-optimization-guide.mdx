### 3. Database Design Optimization ðŸŽ¯

#### Normalization
```sql
-- Before: Products table with repeated category info
CREATE TABLE products (
    id INT PRIMARY KEY,
    name VARCHAR(100),
    category_name VARCHAR(50),
    category_description TEXT
);

-- After: Normalized tables
CREATE TABLE categories (
    id INT PRIMARY KEY,
    name VARCHAR(50),
    description TEXT
);

CREATE TABLE products (
    id INT PRIMARY KEY,
    name VARCHAR(100),
    category_id INT,
    FOREIGN KEY (category_id) REFERENCES categories(id)
);
```

## Real-World Example: E-commerce Optimization ðŸ›ï¸

Let's optimize a product search feature:

```sql
-- Before optimization
SELECT p.*, c.name as category_name
FROM products p
LEFT JOIN categories c ON p.category_id = c.id
WHERE p.name LIKE '%search_term%'
ORDER BY p.created_at DESC;

-- After optimization
CREATE FULLTEXT INDEX idx_product_search ON products(name, description);

SELECT p.id, p.name, p.price, c.name as category_name
FROM products p
INNER JOIN categories c ON p.category_id = c.id
WHERE MATCH(p.name, p.description) AGAINST('search_term' IN BOOLEAN MODE)
ORDER BY p.created_at DESC
LIMIT 20;
```

Improvements:
- âœ… Full-text search instead of LIKE
- âœ… Selected specific columns
- âœ… Added LIMIT clause
- âœ… Changed LEFT JOIN to INNER JOIN
